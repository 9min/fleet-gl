# PRD.md - 제품 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**logi-twin-web** (3D 실시간 물류 관제 대시보드)

### 1.2 프로젝트 목적
대한민국 수도권 V-World 3D 지형 위에서 100대 이상의 물류 배송 차량을 실시간으로 시뮬레이션하고 관제하는 WebGL 기반 대시보드를 구축한다. 단순 UI 구현을 넘어, **대규모 공간 데이터 렌더링 최적화**와 **메인 스레드 병목 방지**에 대한 코어 엔지니어링 역량을 증명하는 포트폴리오/기술 데모 프로젝트이다.

### 1.3 핵심 가치 제안
- 100대 이상의 차량이 동시에 이동하는 장면을 **60fps**로 렌더링
- Web Worker 기반 연산 격리로 **메인 스레드 Zero-blocking** 달성
- V-World 3D Tiles + DEM + 위성영상을 결합한 몰입감 있는 수도권 3D 지형
- 카카오모빌리티 API 기반 실제 도로 경로 위의 사실적 차량 주행 시뮬레이션

## 2. 사용자 페르소나

### 페르소나 A: 기술 면접관 / 채용 담당자
- **목표**: 지원자의 WebGL, 성능 최적화, 시스템 설계 역량을 빠르게 평가
- **행동**: 데모 사이트 접속 → 3D 맵에서 차량 이동 확인 → 인터랙션 테스트 → 코드 리포지토리 확인
- **핵심 기대**: 시각적 임팩트, 부드러운 애니메이션, 깔끔한 코드 구조

### 페르소나 B: 물류 도메인 관계자
- **목표**: 3D 관제 시스템의 실현 가능성과 활용도를 평가
- **행동**: 대시보드 전체 기능 탐색 → 차량 상세 정보 확인 → 통계 패널 분석 → 타임라인 재생
- **핵심 기대**: 직관적 UI, 실시간 배송 현황 파악, 도메인 친화적 정보 제공

### 페르소나 C: 프론트엔드/WebGL 개발자
- **목표**: 기술 구현 방식과 아키텍처 학습
- **행동**: 데모 확인 → DevTools 성능 프로파일링 → GitHub 코드 리뷰
- **핵심 기대**: Web Worker 활용 패턴, Deck.gl 최적화 기법, 상태 관리 전략

## 3. 핵심 기능 명세

### 3.1 3D 지형 렌더링 (V-World)

| 항목 | 상세 |
|---|---|
| 데이터 소스 | V-World Open API (3D Tiles, DEM, 위성/항공 영상) |
| 렌더링 범위 | 수도권 전체 (서울, 경기, 인천 등) |
| 초기 뷰 | 군포 복합물류터미널 중심 조감도 |
| 건물 | 3D Tiles 기반 LoD1/LoD2 건물 모델 |
| 지형 | DEM 기반 고도 데이터로 입체적 지형 표현 |
| 배경 | 위성/항공 영상 타일 오버레이 |
| 카메라 | 자유 회전, 줌, 틸트 지원 |

### 3.2 경로 데이터 파이프라인

| 항목 | 상세 |
|---|---|
| 데이터 소스 | 카카오모빌리티 길찾기 API (다중 경유지) |
| 추출 시점 | **빌드 타임 사전 추출** → JSON 로컬 캐싱 |
| 차량당 경유지 | 약 30개 배송지 순회 경로 |
| 데이터 포맷 | 각 차량별 경로 좌표 배열 (위도, 경도, 타임스탬프) |
| 캐싱 전략 | `public/data/routes/` 디렉토리에 차량별 JSON 파일 저장 |
| 갱신 주기 | 수동 스크립트 실행으로 재추출 |

### 3.3 차량 시뮬레이션 엔진

| 항목 | 상세 |
|---|---|
| 차량 수 | 100대 이상 동시 시뮬레이션 |
| 연산 위치 | **Web Worker** (메인 스레드 격리 필수) |
| 보간 방식 | 경로 좌표 간 선형/큐빅 보간으로 60fps 위치 계산 |
| 데이터 전송 | Worker → Main: `SharedArrayBuffer` 또는 `postMessage` (Transferable) |
| 렌더링 | Deck.gl **TripsLayer** 기반 경로 애니메이션 |
| 시간 제어 | 재생/일시정지, 배속 조절 (1x, 2x, 4x, 8x) |

### 3.4 차량 클릭 상세 정보

| 항목 | 상세 |
|---|---|
| 트리거 | 3D 맵 위 차량 아이콘 클릭/탭 |
| 표시 정보 | 차량 ID, 현재 위치, 배송 진행률, 남은 경유지, 예상 도착 시간 |
| UI | 사이드 패널 또는 팝오버 카드 |
| 경로 하이라이트 | 선택 차량의 전체 경로를 강조 표시 |

### 3.5 타임라인 플레이백

| 항목 | 상세 |
|---|---|
| UI | 화면 하단 타임라인 슬라이더 |
| 기능 | 재생, 일시정지, 배속 조절, 특정 시점 탐색 (Seek) |
| 범위 | 하루 배송 시간대 (예: 06:00 ~ 22:00) |
| 연동 | 슬라이더 조작 시 모든 차량 위치가 해당 시점으로 동기화 |

### 3.6 실시간 통계 패널

| 항목 | 상세 |
|---|---|
| 위치 | 화면 우측 또는 상단 오버레이 패널 |
| 표시 항목 | 총 차량 수, 운행 중/대기/배송 완료 차량 수, 전체 배송 진행률, 총 주행 거리 |
| 업데이트 | 시뮬레이션 시간 변경에 따라 실시간 갱신 |

### 3.7 차량/경로 필터링

| 항목 | 상세 |
|---|---|
| 필터 기준 | 차량 ID, 배송 상태 (운행 중/대기/완료), 경로 지역 |
| UI | 사이드바 또는 드롭다운 필터 컨트롤 |
| 동작 | 필터 적용 시 3D 맵에서 해당 차량/경로만 표시, 통계 패널도 필터 반영 |

## 4. 비기능적 요구사항

### 4.1 성능

| 지표 | 목표 |
|---|---|
| 프레임 레이트 | 100대 차량 동시 렌더링 시 **60fps 유지** (데스크톱 기준) |
| 메인 스레드 블로킹 | Long Task (50ms 초과) **Zero** |
| 초기 로딩 | First Contentful Paint **< 2초** |
| 번들 사이즈 | gzip 기준 초기 로드 **< 500KB** (지도 타일 제외) |
| 메모리 사용 | 브라우저 탭 메모리 **< 512MB** |

### 4.2 브라우저 호환성

| 브라우저 | 지원 수준 |
|---|---|
| Chrome (최신 2버전) | **필수** - 주요 타겟 |
| Edge (최신 2버전) | **필수** |
| Safari (최신 2버전) | **권장** - 태블릿(iPad) 지원 |
| Firefox (최신 2버전) | **선택** |

### 4.3 반응형 디자인

| 디바이스 | 해상도 | 지원 수준 |
|---|---|---|
| 데스크톱 | 1920x1080 이상 | **필수** - 최적화 타겟 |
| 태블릿 | 1024x768 이상 | **필수** - 레이아웃 적응 |
| 모바일 | 375px 이하 | **미지원** |

### 4.4 접근성
- 키보드 내비게이션 지원 (맵 제외 UI 컨트롤)
- 충분한 색상 대비 (WCAG AA 수준)
- 스크린 리더용 통계 정보 접근 가능

### 4.5 보안
- API 키는 환경변수로 관리, 클라이언트 번들에 포함하지 않음
- 경로 데이터는 빌드 타임에 사전 추출하여 정적 파일로 제공
- CSP(Content Security Policy) 헤더 설정

## 5. 기술 스택

| 영역 | 기술 | 선정 이유 |
|---|---|---|
| 프레임워크 | React 18 | Concurrent 렌더링, 광범위한 생태계 |
| 빌드 도구 | Vite | 빠른 HMR, 최적화된 프로덕션 빌드 |
| 언어 | TypeScript | 타입 안정성, 대규모 프로젝트 유지보수성 |
| 3D 렌더링 | Deck.gl | WebGL 기반 대규모 데이터 시각화 특화, TripsLayer 내장 |
| 베이스맵 | react-map-gl (Maplibre GL JS) | V-World 타일 연동, Deck.gl 네이티브 통합 |
| 상태 관리 | Zustand | 경량, 선택적 구독으로 불필요한 리렌더 방지 |
| 병렬 연산 | Web Workers | 보간 연산 메인 스레드 격리 |
| 스타일링 | Tailwind CSS | 유틸리티 퍼스트, 빠른 프로토타이핑 |
| 배포 | Vercel 또는 GitHub Pages | 정적 SPA 무료 배포 |
| 경로 데이터 | 카카오모빌리티 길찾기 API | 다중 경유지 최적화 경로, 국내 도로 데이터 |
| 지형 데이터 | V-World Open API | 국내 3D Tiles, DEM, 위성영상 공식 제공 |

## 6. 데이터 흐름 요약

```
[빌드 타임]
카카오모빌리티 API → 경로 추출 스크립트 → JSON 캐시 파일 (public/data/routes/)

[런타임]
JSON 캐시 로드 → Web Worker (좌표 보간 연산) → Main Thread (Deck.gl 레이어 업데이트)
                                                  ↕
                                          Zustand (UI 상태만)
                                                  ↕
                                      React 컴포넌트 (통계, 필터, 패널)
```
