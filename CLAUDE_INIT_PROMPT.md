**[프로젝트 초기 개요]**
* 프로젝트 이름: logi-twin-web (3D 실시간 물류 관제 대시보드)
* 핵심 목표: 대한민국 지형(V-World) 기반의 수십 대 물류 배송 차량 실시간 3D WebGL 관제 대시보드 구축. 단순 UI 구현을 넘어, 대규모 공간 데이터 렌더링 최적화와 메인 스레드 병목을 방지하는 코어 엔지니어링 역량 증명이 목적입니다.
* 주요 기능: 
  1. 브이월드(V-World) 3D 타일 기반의 군포 복합물류터미널 중심 지형 및 건물 렌더링
  2. 카카오모빌리티 API를 활용한 다중 경유지(30여 개) 최적화 경로 데이터 추출 및 로컬 캐싱
  3. Web Worker를 활용한 60fps 차량 이동 좌표(보간) 실시간 계산 및 Deck.gl TripsLayer 렌더링
* 기술 스택: React 18, Vite, TypeScript, Deck.gl, Zustand, Web Workers (기타 필요한 툴은 추가 추천 요망)

**[아키텍처 핵심 제약 및 설계 방향] (문서 작성 시 반드시 반영할 것)**
* 상태 관리 제약: 좌표 데이터가 초당 60회 변경되더라도 전체 페이지 리렌더링을 유발해서는 안 되며, Deck.gl 레이어만 독립적으로 업데이트되어야 합니다.
* 연산 격리: 차량 위치 보간(Interpolation) 연산 로직은 무조건 Web Worker로 격리해야 합니다.
* 모듈화: API 통신 로직, WebGL 렌더링 로직, Web Worker 로직은 각각 별도의 디렉토리로 명확히 분리하십시오.
